
# User Requirements

## Introduction

This document outlines the user requirements for the Spa application. It provides a detailed description of the functionalities and features that the system must support to meet the needs of its users.

## Bookings 
1. **User Registration and Authentication**
   - Users must be able to booking an appointment without creating an account. 
   - Users should be able to receive a confirmation email after booking an appointment.
   - The system should flow the GDPR law according which data should be gathered and stored securely.

2. **Appointment Scheduling**
    - Users must be able to view available time slots for appointments.
    - Some date should be blocked for booking due to holidays or maintenance. But there are some dates that should be open for booking even if they are holidays. ( like Valentine's Day, Mother's Day, New Year's eve, first May).
    - there are three packages for booking: could "kall", hot "Värm" and peace & quiet "lugn o ro". Each  package has different services and prices. Users should be able to choose the package that suits  their needs and budget. When an package is booked, than this package should be blocked for booking for the same date and time.
    - Users should be able to cancel their appointment in 20 seconds after booking it. After 20 seconds, the appointment cann't be canceled by the user, but they can contact the support team to cancel it.
    - The email confirmation should include the details of the appointment, such as the date, time, package, and any special instructions, as well as a telephone number for support.

3. **Employee & Admin Management**
    - Employees should be able to view all appointments scheduled for the day. That includes the date, time, package, and customer details. Without the ability to edit or cancel appointments or to see the full price.
    - Admins should have the ability to manage appointments, including the ability to view, edit, and cancel appointments as needed. Admins should also be able to manage employee schedules.
    - Admins should be able to manage the packages, including the ability to add, edit, and remove packages as needed.
    - Admins should be able to see all the appointments, including the full price for the hole day.
    - Employees should not have access to the full price of the packages, but they should be able to see the price for the services included in the package. This is to ensure that employees are aware of the services they are providing without being influenced by the total price of the package.

## InterFaces 
Admins should have tow different interfaces: one for managing appointments and see all the details of the appointments. 
The other interface should be for managing the packages, including the ability to add, edit, and remove packages as needed. As well as managing the employees schedules.

Employees should have an interface to view all appointments scheduled for the day, including the date, time, package, and customer name and contact details.

Customers should have an interface to book appointments, view available time slots, and receive confirmation emails. 

Landing page should be simple and clear, the costumers should be able to see what the spa offers and how to book an appointment. The landing page should also include contact information for the spa, such as a phone number and email address, as well as links to social media profiles.
